# Preview Kit术语

_Source: https://developer.huawei.com/consumer/cn/doc/harmonyos-guides/preview-kit-glossary_

A

[h2]App Preload；应用预加载

系统提供的一种应用启动加速能力。在用户点击启动应用前，系统提前加载应用进程，加快应用启动速度。

[h2]Atomic Time；原子耗时

文件打开过程中，不能被外部打断的操作花费的时间。

C

[h2]Cache Eviction Policy；缓存淘汰策略

当缓存使用量超过配置上限时，系统自动执行的缓存清理机制。策略会根据缓存对象的权重、访问频率等指标决定淘汰顺序，优先释放低优先级的缓存对象，为新的缓存对象腾出空间。

[h2]Cache Object；缓存对象

存储在文件缓存系统中的数据单元。可以是简单的二进制数据，也可以是通过序列化函数处理的复杂类型对象，支持通过缓存键进行快速查询和访问。

F

[h2]File Cache Boost；文件缓存加速

通过缓存优化文件操作性能的机制。应用通过该机制将程序对象数据缓存到本地存储，在文件打开和浏览场景中优化性能，减少程序对象数据二次构造时间，优化文件操作性能。

[h2]File Preload；文件预加载

系统根据用户行为预测即将打开的文件，并提前通知应用将文件内容加载到内存中的能力。通过提前加载，当用户实际打开文件时可直接使用预加载内容，显著缩短文件打开时间。

[h2]File Scan Boost；文件扫描加速

针对复杂文件的文件加速预处理。通过异步扫描文件内容并预先处理，为文件打开加速提供决策依据。

M

[h2]Max Atomic Time；最大原子耗时

文件打开全流程，所有文件操作中最大的原子耗时。在文件扫描加速流程中，应用通过上报最大原子耗时，帮助系统评估文件加速任务的性能开销，决策被扫描文件是否加速打开。

O

[h2]Open File Boost；文件打开加速

一种大文件打开加速机制，通过预测用户可能打开的文件并提前预加载，缩短大文件的打开时间。

P

[h2]Preload Hit；预加载命中

当用户打开的文件恰好是系统之前预加载的文件时，应用通过通知接口告知系统预加载命中。命中信息用于优化系统的预加载预测算法，提高后续预测的准确性。
