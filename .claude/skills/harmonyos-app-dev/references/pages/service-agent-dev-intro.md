# 鸿蒙智能体

_Source: https://developer.huawei.com/consumer/cn/doc/service/developer-guide-0000002469667881_

鸿蒙智能体是运行在HarmonyOS设备上的AI智能体，通过小艺开放平台开发、调试和分发。开发者可以创建智能体，配置其能力，然后发布到智能体市场供用户使用。

## 智能体分类

### 单Agent（LLM模式）
基于大模型的智能体编排方式。开发者按需选择大模型，根据业务逻辑编写提示词，以LLM为理解中枢，结合意图识别、工具调用、对话上下文，动态选择插件、工作流，响应用户需求。适用于简单对话、知识问答、基础内容生成等场景。

### 工作流模式
基于规则化流程的智能体编排方式。开发者将复杂任务拆解为有序的规则化步骤（如数据获取、处理、执行），串联插件、大模型、条件分支、代码块等组件实现自动化执行流程。适用于需多步骤协同、逻辑复杂、业务多样性的场景。

### A2A模式
三方智能体接入小艺开放平台的高效编排方式。开发者可通过该模式基于鸿蒙Agent通信协议快速、便捷地将成熟的第三方智能体对接至小艺开放平台，实现分发与调用。该模式适用于同时具备鸿蒙端应用与云侧智能体能力的企业开发者。

### OpenClaw模式
开放灵活的智能体接入与构建方式，开发者可通过该模式接入OpenClaw工具，快速创建个性化智能体。该模式适用于个性化助手、自动化服务、场景化应用等多样化需求。
